Theoretical Foundation (Cơ sở lý thuyết)
Hash-based Vertex-Home Edge-Cut Partitioning HomeNode(V) = Hash(V) % K where K = number of sites Strategy: All outgoing edges of vertex V are stored at HomeNode(V). This co-locates adjacency data for efficient local lookups. Edge-Cut Ratio = (Cut Edges / Total Edges) × 100% - A "cut edge" is one where source and destination have different home nodes. - Lower ratio = better partitioning = less cross-shard communication. Vertex Replication Factor = Σ(partitions each vertex appears in) / Total Vertices - Factor of 1.0 = each vertex appears in exactly one partition (no replication). - Higher factor = more replication overhead but potentially faster local lookups. Cost Model (Ch.4 - Query Processing) Total_Cost = I/O_Cost + CPU_Cost + Communication_Cost - Communication_Cost is the dominant bottleneck in distributed systems. - Each cross-shard path expansion = 1 HTTP network message. - Goal: Minimize network messages while maintaining detection accuracy.
